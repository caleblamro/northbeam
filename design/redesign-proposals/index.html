Design exploration · July 2026

Page redesign proposals

Twenty layout concepts across eight surfaces — round 1 (home, lists, record detail: shipped) and round 2 (reports, dashboards, migrate, setup, sign-in). All keep the existing token system — near-black brand, single indigo accent, Inter, hairline borders — and only rework composition, hierarchy, and density. Same seed data everywhere so comparisons are fair.

Use the sun/moon toggle in the topbar of any mockup — every concept is themed for light and dark.

Home dashboard

Fixes: the actionable panel being buried; the page tells you what to do, not just what exists. Open mockup →

Views — one library for everything

supersedes the separate Reports + Dashboards pages below

There are no dashboards or reports in this product — there are saved views (list · report · dashboard · record layout). One surface should hold all of them: browse, preview, filter, create, manage defaults.

V1The library Every saved view is a rich preview card in one filterable library — type-aware mini-previews (table skeletons, bar lists, widget grids, form skeletons), default stars, scope pills, owners, and a New-view menu covering all four types + AI. Fixes: the artificial reports/dashboards split; carries the rich-tiles direction further. Open mockup → V2Explorer Organized the way the data model is: an object rail (with per-object type-mix dots) beside compact type-grouped rows, each with an inline micro-preview and manage actions. Fixes: view sprawl across objects; mirrors how views actually attach to objects. Open mockup → V3Workbench Split master-detail for views: searchable grouped list on the left, a live render of the selected view on the right with Set default / Edit / Open — browse and preview without navigating. Fixes: information-poor tiles; you see the actual view before opening it. Open mockup →

Roles & permissions

currently: two save models on one screen, opacity-only inheritance cues, seed-locked toggles that look editable, members invisible

The two-pane editor works but can't be trusted at a glance: batched Save beside instant-save checkboxes, inherited cells at 60% opacity, system grants that silently can't be revoked, and no way to see who holds a role.

P1Role studio The two-pane made trustworthy: one visible save model with dirty markers, members managed in context ("delete requires zero members — move them here"), explicit Inherits/Override chips on the object grid, and AI tools as a section of the role itself. Fixes: all five audit findings in one coherent surface. Open mockup → P2Access matrix Roles as columns, permissions as rows — "who can do X" in one glance. Legend-backed cell states (granted / not / seed-locked), CRUD letter clusters per object, instant saves declared in the footer with an audit-log link. Fixes: auditability; makes seed-granted immutability honest with lock glyphs. Open mockup → P3Personas Roles as people first: cards with member avatars and capability meters, a focused drill-in editor, and a "Preview the workspace as this role" simulation affordance. Fixes: the invisible-members problem; adds role simulation. Open mockup →
